Section 02

Mode de fonctionnement interne

Le groupe a organisé son travail selon une répartition claire des responsabilités, chaque membre se voyant attribuer un ou plusieurs modules de l'application en fonction de ses affinités techniques et de sa filière.

La coordination a été assurée via Git/GitHub pour la gestion du code source, avec des commits réguliers de chaque contributeur. Les échanges et le suivi des tâches ont été organisés en dehors des séances encadrées, en s'appuyant sur les sessions de travail prévues au calendrier (01, 03, 04, 08 et 09 juin 2026).

Démarche suivie

  • Phase 1 – Analyse : lecture du cahier des charges, identification des modules, définition des tables SQL.
  • Phase 2 – Conception : modélisation de la base de données, maquettage des interfaces.
  • Phase 3 – Développement : développement Frontend en parallèle du Backend, intégration progressive.
  • Phase 4 – Intégration : liaison Frontend–Backend, tests des fonctionnalités principales.
  • Phase 5 – Livraison : rédaction du rapport, finalisation du fichier .sql, dépôt GitHub.

Section 03

Description de la conception

Architecture générale

L'application suit une architecture client-serveur : le frontend (HTML/CSS/JavaScript) communique avec le backend Python/Flask via des requêtes HTTP. Le backend gère la logique métier et interagit avec la base de données MySQL. 

Navigateur (HTML/CSS/JS)
       │
       │  HTTP (fetch / formulaires)
       ▼
  Serveur Flask (Python)
       │
       │  SQL (PyMySQL / SQLAlchemy)
       ▼
  Base de données MySQL

Modules de l'application

  • 👤
    Gestion des comptes et profils
    Inscription, connexion, réinitialisation du mot de passe, édition du profil (filière, compétences, disponibilités, photo).
  • 🤝
    Mise en correspondance (Matching)
    Publication d'offres et demandes de mentorat, algorithme de matching basé sur les compétences, filières et disponibilités, affichage du score de compatibilité.
  • 💬
    Messagerie instantanée
    Conversations entre utilisateurs, historique des messages, notifications en temps réel à la réception d'un message.

Structure de la base de données

La base de données MySQL est composée des tables suivantes :

utilisateurs filieres_etudes niveaux_etudes matieres competences_utilisateur difficultes_utilisateur disponibilites offre_mentorat demande_mentorat correspondances conversations messages notifications parametres participants_conversation

Le schéma complet (CREATE TABLE, clés primaires et étrangères, contraintes) est disponible dans le fichier ifri_mentorlink.sql à la racine du dépôt.

Section 04

Instructions de déploiement

Prérequis : Python 3.10+, MySQL 8+, Git.

  1. Cloner le dépôt
    git clone https://github.com/[ORG]/PIL1_2526_61.git
cd PIL1_2526_61
  2. Créer et activer un environnement virtuel Python
    python -m venv venv
# Windows :
venv\Scripts\activate
# macOS/Linux :
source venv/bin/activate
  3. Installer les dépendances
    cd backend
pip install -r requirements.txt
  4. Créer la base de données et importer la structure
    Dans MySQL : créer une base mentorlink_db, puis importer : 
    mysql -u root -p ifri_mentorlink_db < ../database.sql
  5. Configurer la connexion à la base de données
    Modifier les variables dans backend/app.py (ou un fichier .env) : 
    DB_HOST = "localhost"
DB_USER = "root"
DB_PASSWORD = "votre_mot_de_passe"
DB_NAME = "ifri_mentorlink_db"
  6. Lancer le serveur Flask
    python app.py
    Le serveur démarre sur http://localhost:5000.
  7. Accéder à l'application
    Ouvrir un navigateur et naviguer vers http://localhost:5000.

Section 05

Manuel d'utilisation

Créer un compte

Accéder à la page Inscription. Renseigner : nom, prénom, adresse e-mail, numéro de téléphone, filière, niveau et mot de passe. Choisir ses compétences (points forts) et ses difficultés (points faibles). Valider le formulaire.

Se connecter

Depuis la page Connexion, saisir son adresse e-mail (ou numéro de téléphone) et son mot de passe. En cas d'oubli, utiliser le lien « Mot de passe oublié » pour recevoir un lien de réinitialisation.

Publier une offre ou une demande de mentorat

Depuis le Dashboard, cliquer sur « Nouvelle offre » ou « Nouvelle demande ». Renseigner les matières concernées, les disponibilités et le format souhaité (présentiel / en ligne). Soumettre pour que l'algorithme de matching propose automatiquement des correspondances.

Consulter les correspondances

La section Matching affiche les profils compatibles avec un score de compatibilité, les matières en commun et les disponibilités partagées. Cliquer sur un profil pour envoyer un message ou une demande de mentorat.

Messagerie

Accéder à l'onglet Messages pour consulter et envoyer des messages textuels. Les nouvelles conversations apparaissent dans la liste à gauche. Les notifications en temps réel signalent l'arrivée de nouveaux messages.

Paramètres du compte

Depuis la page Paramètres, modifier les informations de profil (photo, bio, disponibilités, compétences, difficultés) et les préférences de notification.

Section 06

Difficultés rencontrées

Au cours du développement de IFRI_MentorLink, le groupe a rencontré plusieurs défis techniques et organisationnels.

Conception de la base de données : La modélisation des relations entre les nombreuses entités (utilisateurs, compétences, disponibilités, offres, demandes, correspondances, conversations) a nécessité plusieurs itérations pour garantir la cohérence des données et éviter les redondances. La définition précise des clés étrangères et des contraintes d'intégrité référentielle a été particulièrement exigeante.
Liaison Frontend–Backend : La synchronisation entre les interfaces HTML et les routes Flask, notamment pour la gestion des sessions utilisateur et la transmission sécurisée des données via les formulaires, a demandé un effort d'intégration important.
Coordination de l'équipe : Travailler en parallèle sur plusieurs modules a nécessité une discipline stricte dans l'utilisation de Git pour éviter les conflits de fusion (merge conflicts) et maintenir la cohérence du code entre les branches.
